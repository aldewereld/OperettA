Monitor Tool

The monitor tool is a process that allows end-users to visually analyse the events being interchanged on the ALIVE system. The monitor tool is able to read events from the event bus and from the event log, allowing both the analysis of events generated at run time and the analysis of past events.

Step Through

Configuration

Once the monitor tool has been started, users can specify the locations of the different event bus and event log components to read events from.

To add a new event bus, go to event bus locations view and right-click anywhere on the view. A pop-up menu appears. Select add event bus option from the menu. A new menu, asking for the location (i.e. IP address or URI) of the event bus to add appears. Enter the location of the event bus to add (i.e. 127.0.0.1 or localhost for a local event bus) and press OK button. This will effectively add the new event bus to the event buses group. The location will remain there, until it is explicitly removed by right-clicking on it and using the remove command.

To add a new event log, follow the process of adding a new event bus, but selecting the event log option. Notice the event log is added to the Event logs group. Event log locations are also persistent. Right-clicking on an event log and using remove option will effectively remove the event log from the set of event logs the monitor tool reads events from.

Adding a new event log

Running

To start the monitor tool enable both event bus locations and event list views on the ALIVE bundle.

This can be done by:

Clicking on Window menu

Going to show view menu

Selecting both event bus locations and event list views. If they are not available, selecting other menu. Then, from other, inside monitoring folder, selecting both event bus locations and event list views.

Monitoring views

Once the monitor tool is running, go to the event bus locations view. This view shows two groups, event bus and event log that contain the event bus and event log components the monitor tool can listen to. For instructions on how to add new event bus or event log components refer to monitor tool configuration section.

In order to start receiving events from an event bus or event log users must double click on one of the locations, effectively connecting to it.

Once the monitor tool starts receiving events, it will add them to the event list view. Users can sort events based on any of the columns by double clicking on it. For instance, in order to see the events ordered by Asserter, double click on Asserter column. It will effectively sort the events by asserter name in ascending order. Double-clicking again on the column will effectively sort events by asserter name, descending order.

Users can filter the events the event list shows as well. To filter, specify a search pattern on the Search field. The search pattern contains a regular expression that is matched against Asserter name and Timestamp column values. If a match occurs, the event is shown in the list, otherwise it is hidden.

Event list view

Users can visualize events on the event view as well. To do it, double click on a value of the Local key column. This will effectively show an event model, with the concepts related to the event, and their values in a graph-like structure. Concepts can be freely moved around the event view (by dragging and dropping them) to make the graph visually clearer and easier to understand.
