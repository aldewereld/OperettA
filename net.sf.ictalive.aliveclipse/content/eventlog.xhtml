Event Log

The event log is a back-end process able to capture relevant events from the event bus and store them in persistent data storage. The event log is also able to read events from the persistent store and provide them to other components (such as the monitor tool).

The event log process relies on a Cassandra (http://en.wikipedia.org/wiki/Apache_Cassandra) for storing the events. Cassandra is shipped along the ALIVE bundle installation, so a local event log process can be easily configured and set to run.

Step Through

Configuration

Event log configuration can be specified via the runtime dashboard. There, users can specify the location of the Cassandra instance to be used by the event-log process, that will run always as a local back-end process. By default the Cassandra instance is running on the local machine.

Running

Event log can be started via the Start Event Log button on the runtime dashboard. Notice that, having a back-end event log process running is not mandatory for the ALIVE framework to work. Users are encouraged to start the event log process only if they want their events to be stored on persistent data storage.
